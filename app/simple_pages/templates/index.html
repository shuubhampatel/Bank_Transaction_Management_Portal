Pylint

Pylint is a static code analysis tool written in Python for the purpose of analysis. It may check for programming flaws, implement coding standards best practices, discover code violations, and advise reworking. It is totally customisable by the user and has built-in specific routines for controlling particular errors and alarms within the user's code.

AAA testing

The AAA pattern is a pattern for structuring tests. It breaks each test down into three parts – Arrange, Act, and Assert – where each part is a step leading to the next. The arrange step sets up the test’s input values. The act step prompts the primary function being tested. And finally, the assert step verifies that the output of the function is what was expected.

OOPs Concepts

Object-Oriented Programming (OOP) uses "objects" to model realworld objects. Object-Oriented Programming (OOP) consist of following some important concepts referred to as the OOPS concepts.

  • Encapsulation
  • Inheritance
  • Polymorphism
  • Abstraction

SOLID principles

SOLID is an acronym that stands for five object-oriented design principles that help you make the software more understandable, easier to maintain, and easier to extend.

  • Single-responsibility
  • Open-closed
  • Liskov Substitution
  • Interface Segregation
  • Dependency Inversion

Git

GIT is Free and open source Version Control System (VCS) is a software that helps software developers to work together and maintain a complete history of their work and also it comes under distributed version control system (DVCS).

Docker

Docker is a software platform that makes it easier to create, execute, manage, and distribute apps. It accomplishes this by virtualizing the operating system of the machine on which it is installed.

Python/ Flask

Flask is a Python-based web application framework. It belongs to the micro-framework category, which means it doesn't rely on external libraries. The Werkzeug WSGI toolkit and the Jinja2 template engine are the foundations of Flask.

CI/CD

A continuous integration/continuous delivery (CI/CD) pipeline is a set of procedures that must be followed in order to deliver a new version of software. Continuous integration and continuous delivery (CI/CD) pipelines are a DevOps or site reliability engineering (SRE) activity designed to optimize software delivery.
