Lets read about OOPs concepts

Object-Oriented Programming (OOP) uses "objects" to model realworld objects. Object-Oriented Programming (OOP) consist of following some important concepts referred to as the OOPS concepts.

Lets understand this concept with the calculator program

  • Encapsulation:In general terms Wrapping up of a data and functions into single unit is known as encapsulation.Here are the code example and we can clearly see that when we create Instance of class Division it will bind and store all the data and functionally to the calculation variable to perform get_result() method. 
    from calculator.calculations import Addition, Subtraction, Multiplication, Division


class Calculator:
    """ This is the default result property"""

    @staticmethod
    def divide(tuple_list):
        """ This is the divide method"""
        calculation = Division.create(tuple_list)
        return calculation.get_result()

    Here method create() is wrapped in one unit so above is a one example of Encapsulation from calculator app.

  • Inheritance: Inheritance is the process/ Mechanism, by which class can acquire the properties and methods of another class. in simple words deriving a new class from an old class is called inheritance.Let's understand more with code so below is the code example that shows Inheritance in calculator app.
    class Calculation:
    """ calculation abstract base class"""

    # pylint: disable=too-few-public-methods
    def __init__(self, tuple_list: tuple):
        """ constructor method"""
        self.values = Calculation.convert_args_to_tuple_of_float(tuple_list)

    @classmethod
    def create(cls, tuple_list: tuple):
        """ factory method"""
        return cls(tuple_list)

    @staticmethod
    def convert_args_to_tuple_of_float(tuple_list):
        """ standardize values to list of floats"""
        list_values_float = []
        for item in tuple_list:
            list_values_float.append(float(item))
        return tuple(list_values_float)

So that's how Inheritance works in calculator app.

  • Abstraction: Abstraction is a concept in object-oriented programming that "shows" just the most important features while "hiding" the others. The basic goal of abstraction is to keep people aside from unneeded elements. Below code gives example of Abstraction by "hiding" features from calculations class. so in the hightlighted line you can see that we are importing methods Addition, Subtraction, Multiplication, Division from calculations class that means we shows just divide, add, subtract, multiply methods and we are hiding hows the calculation works. 
    """ This is the Calculator Class"""
from calculator.calculations import Addition, Subtraction, Multiplication, Division


class Calculator:
    """ This is the default result property"""

    @staticmethod
    def divide(tuple_list):
        """ This is the divide method"""
        calculation = Division.create(tuple_list)
        return calculation.get_result()

    @staticmethod
    def add(tuple_list):
        """ This is the add method"""
        # Call the static method add to return the sum and set it to the calculator result property
        calculation = Addition.create(tuple_list)
        return calculation.get_result()

    @staticmethod
    def subtract(tuple_list):
        """ This is the subtract method"""
        calculation = Subtraction.create(tuple_list)
        return calculation.get_result()

    @staticmethod
    def multiply(tuple_list):
        """ This is the multiply method"""
        calculation = Multiplication.create(tuple_list)
        return calculation.get_result()

class Addition(Calculation):
    """ calculation addition class"""

    def get_result(self):
        """get the addition results"""
        sum_of_values = 0.0

            sum_of_values = Add.add(value, sum_of_values)
        return sum_of_values
