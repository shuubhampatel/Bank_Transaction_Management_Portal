Lets read about Pylint

Pylint is a static code analysis tool written in Python for the purpose of analysis. It may check for programming flaws, implement coding standards best practices, discover code violations, and advise reworking. It is totally customisable by the user and has built-in specific routines for controlling particular errors and alarms within the user's code.

to install pylint on windows or mac machine according to documentation you should write
pip install pylint

Pylint will display a number of messages as it analyzes the code and it can also be used for displaying some statistics about the number of warnings and errors found in different files. The messages are classified under various categories such as errors and warnings.

Following are the General definitions:

  • OOP terms with definitions:
    Object-Oriented Programming (OOP) uses "objects" to model realworld objects. Object-Oriented Programming (OOP) consist of following some important concepts referred to as the OOPS concepts.Please click here to learn this terms by Calculator Program examples
    • Encapsulation:Wrapping up of a data and functions into single unit is known as encapsulation.

    • Inheritance: Inheritance is the process/ Mechanism, by which class can acquire the properties and methods of another class. in simple words deriving a new class from an old class is called inheritance.

    • Polymorphism: Polymorphism allows a single name to be used for more than one related purpose.The concept of polymorphism is characterized by the idea of 'one interface, multiple methods'.

    • Abstraction: Abstraction is a concept in object-oriented programming that "shows" just the most important features while "hiding" the others. The basic goal of abstraction is to keep people aside from unneeded elements.
  • Instantiation:
    Object creation in Python is a two-step process. first step is creating objects and second is initialization the object. Python uses __new__ method for object creation and __init__ for initialization.
    During the object instantiation process if the class does not define these methods, they are inherited from the objectbase class.
  • Object:
    An object is an instance of a class.An object is a component of a program that knows how to interact with other pieces of the program.
  • Class:
    A class is a template that specifies the attributes and behavior of things or objects.A class is a prototype from which objects are created. It defines attributes and methods.
  • Namespace:
    It defines a scope for the identifiers that are used in a program. in other words we can think of a namespace as a dictionary in which the keys are the object names and the values are the objects themselves. Each key-value pair maps a name to its corresponding object.
  • Constructor:
    Constructors are generally used for instantiating an object. The task of constructors is to initialize(assign values) to the data members of the class when an object of the class is created. in python __init__() method is called the constructor.

  • Fixture:
    Fixtures are functions, which will run before each test function to which it is applied. Fixtures are used to feed some data to the tests such as database connections, URLs to test and some sort of input data. But Point to note is fixture only works with one test file but if you want to use that fixture in another test file. To make a fixture available to multiple test files, we have to define the fixture function in a file called conftest.py.

    Check that your code is working as expected by creating and running unit tests. It's called unit testing because you break down the functionality of your program into discrete testable behaviors that you can test as individual units.
